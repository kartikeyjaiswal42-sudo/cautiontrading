# cautiontrading/ — full history

**CautionTrading** (built 2026-06-10) — live technical-indicator alert engine for a paying client who trades futures/options on **Delta Exchange India** (modeled on his TradingView alert screen: RSI crossing on 5m). User is non-technical and fully dependent on Claude for bugs. Plain Node.js (v25) + Express, no other deps, no build step. Run: `npm start` (or double-click `Start CautionTrading.command`) → http://localhost:8899.
- **Data:** Delta India public API (`https://api.india.delta.exchange`) — `/v2/products` (1,111 symbols, paginated via `meta.after`) + `/v2/history/candles` (returns newest-first; server sorts ascending). No API key. Symbols like USOIL are TradingView-only, not available.
- **Engine (`server.js` + `engine.js`):** checks every **1 second** (user explicitly rejected 10s; `CHECK_INTERVAL_MS` env overridable). Alerts grouped by symbol+interval share one candle fetch (700ms cache) — stays far under Delta's ~10k/5min rate limit. Alert = {symbol, resolution, left operand (indicator spec), op (cross_up/cross_down/cross/gt/lt), right (fixed value OR another indicator — e.g. EMA9 × EMA21), trigger (once / once_per_bar [closed candles only] / every [60s cooldown]), message, expiresAt, channels{app,telegram}}. gt/lt are edge-triggered via per-alert runtime state (`lastAbove`); runtime state resets on edit/re-enable. "once" alerts → status `fired`; toggle re-arms.
- **Indicators (`indicators.js`):** 25 types (price/volume/RSI/SMA/EMA/WMA/HMA/VWAP/MACD/BB/Stoch/StochRSI/ATR/ADX/CCI/W%R/MFI/ROC/Mom/OBV/Supertrend/PSAR/Ichimoku/Keltner/Donchian) with params/source/multi-output registry that **auto-builds the UI form** (`/api/meta`). Wilder RMA for RSI/ATR/ADX — RSI verified against Wilder's canonical 70.46. **GOTCHA fixed:** rolling-sum SMA gets permanently NaN-poisoned by indicator warm-up NaNs (broke Stoch %K) — sma() must be windowed.
- **Notifications:** SSE (`/api/stream`) → full-screen flashing red alarm overlay + WebAudio siren (square osc + 3Hz LFO; needs the "Enable sound" click first — browser autoplay policy) + desktop Notification; **Telegram** via bot API (settings store token+chatId; `/api/telegram/detect-chat` auto-finds chat id from getUpdates after user messages the bot; test button). Fired log capped 300 in `data/store.json` (JSON store, debounced atomic writes).
- **UI (`public/`) — full TradingView-style terminal (rebuilt 2026-06-10 after user feedback "UI/UX very bad, only one stock shown, want everything like Delta/TradingView so client sees everything on my website and only goes to Delta to execute"):** live candlestick **chart via `lightweight-charts` 5.2** (TradingView's own OSS lib, vendored to `public/vendor/`, v5 API: `chart.addSeries(LightweightCharts.CandlestickSeries, opts, paneIndex)`, panes for oscillators via paneIndex 1 + `chart.panes()[1].setHeight()`, `autoSize:true`). Chart = candles + volume histogram + OHLC legend w/ crosshair readout, refreshed 2s via new **POST `/api/series`** (returns candles + computed indicator series; NaN→null for JSON). Topbar: symbol button → **symbol-browser modal** (search + Perpetuals/Futures/Options/All tabs, live last-price+24h% from new **GET `/api/tickers`** proxy cached 10s, sorted by turnover, 250 rows shown), live quote, interval pills. Right sidebar: Alerts / Triggered Log tabs. **Clicking an alert card plots its indicator on the chart** (+dashed amber target line via `createPriceLine`); while the create-alert dialog is open the chart **live-previews** the condition being built. Indicator dropdown replaced with **searchable grouped picker** (`makeIndicatorPicker`: Price&Volume/Trend/Momentum/Volatility/Volume-based groups). Overlay-vs-pane split via `OVERLAY` set in app.js; supertrend `direction` output excluded from charting; `hist` outputs render as ±colored histogram. Verified end-to-end via Playwright (`channel:'chrome'`, script must live in ntpc-singrauli-sim/ for its node_modules): chart renders, symbol modal w/ live prices, picker, fire → alarm overlay → card click → EMA overlay plotted; zero console errors.
- **Monetization context:** client suggested user could sell this as a subscription. Current build is single-user/local; subscriptions would need hosting (Render/Railway — plain Node, no DB), accounts, payments — phase 2 if asked.
